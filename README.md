# GCP Professional Machine Learning Engineer Study Plan

A comprehensive 18-day study plan for the Google Cloud Professional Machine Learning Engineer certification exam.

## Exam Details

| Detail | Information |
|--------|-------------|
| **Exam** | Google Cloud Professional Machine Learning Engineer |
| **Duration** | 2 hours |
| **Questions** | ~60 multiple choice/select |
| **Registration** | [Google Cloud Certification](https://cloud.google.com/learn/certification/machine-learning-engineer) |

## Study Plan Structure

This study plan covers 18 days of content spread across 4 weeks (with weekends off):

### Week 1 (Dec 28 - Jan 3)
- **Day 1**: Introduction to AI and ML on Google Cloud
- **Day 2**: ML Workflow and Generative AI
- **Day 3**: Prepare Data for ML APIs (Part 1)
- **Day 4**: Prepare Data for ML APIs (Part 2)
- **Day 5**: Notebooks and BigQuery ML
- **Day 6**: BigQuery ML Advanced

### Week 2 (Jan 6 - Jan 10)
- **Day 7**: Engineer Data for Predictive Modeling
- **Day 8**: Feature Engineering
- **Day 9**: TensorFlow on Google Cloud
- **Day 10**: Production Machine Learning Systems
- **Day 11**: MLOps Fundamentals

### Week 3 (Jan 13 - Jan 17)
- **Day 12**: MLOps for GenAI and Model Evaluation
- **Day 13**: Building Generative AI Applications
- **Day 14**: ML Pipelines and Orchestration
- **Day 15**: Responsible AI
- **Day 16**: Monitoring and Review (Sections 1-3)

### Week 4 (Jan 20 - Jan 21)
- **Day 17**: Review Sections 4-6
- **Day 18**: Final Review and Exam Prep

## Exam Sections Covered

| Section | Topic | Weight |
|---------|-------|--------|
| 1 | Architecting Low-Code ML Solutions | ~12% |
| 2 | Collaborating Within and Across Teams | ~16% |
| 3 | Scaling Prototypes into ML Models | ~18% |
| 4 | Serving and Scaling Models | ~19% |
| 5 | Automating and Orchestrating ML Pipelines | ~18% |
| 6 | Monitoring ML Solutions | ~17% |

## How to Use This Study Plan

1. Each day's markdown file contains:
   - Direct links to all videos, labs, quizzes, and readings
   - Time estimates for each activity
   - Key concepts and code examples
   - End-of-day checklist to track progress

2. Open the day's file and work through the activities in order

3. Use the checkboxes to track your progress

4. Take notes in the designated notes section of each file
